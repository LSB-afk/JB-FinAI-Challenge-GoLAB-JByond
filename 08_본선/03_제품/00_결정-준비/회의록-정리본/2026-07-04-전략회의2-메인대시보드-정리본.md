## 참석/맥락

2:25 전략회의(업무 큰 그림·인풋/프로세스/아웃풋·나선형)에 이어, 저녁 6:03에 **메인 대시보드의 실제 화면 구성·인터랙션**을 손그림으로 확정했다. 김민주가 화면 흐름을 주도하고, 김주용이 에이전트 계층·데이터 구조를, 이승보가 구현 관점(트랜지션·취합·토큰)을 보탰다. 회의는 김민주의 **"8분 안에 피그마로 만들어서 보내드릴게요"**로 마무리 — 디자이너 실작업이 이 회의 결정을 받아 착수한다.

## 핵심 요약 (3줄)

1. **메인 = 업무 보드**: RM이 진입 즉시 "오늘 할 일(케이스 N건, 진행 전/중/후 카운트, 우선순위)"을 보고, 케이스를 눌러 들어가 **엔터/키보드만으로** 하위 에이전트 실행을 승인하며 밀어낸다. **마우스 절대 금지**(한 손 노트·한 손 조작).
2. **에이전트 계층 실행**: 케이스 담당 에이전트가 하위 에이전트들에게 스킬/MCP/플러그인/내부규정을 위임 → 각자 MD 생성 → **최종 취합 MD 1개**. 실행 중엔 딤드/로딩, 완료 시 알림(띠링) → 담당자가 돌아와 근거·기대값 보고 승인, 반려 시 재순환.
3. **결과물 = MD 문서 뷰**: 통합본 + 개별 에이전트 MD를 **옵시디언/노션식 링크**(통합→개별, `@에이전트`→에이전트 화면)로 연결하고, 플로팅 모달/탭/사이드로 열람. 사용 스킬·출처는 하단. **2개 계열사만** 시연 범위로 확정.

## 공유·결정

1. **진입 화면 = 업무 보드(메인)**. RM은 들어오자마자 오늘 할 일(무엇을·몇 개)을 본다. 케이스가 여러 개 → **우선순위**로 정렬. 김민주 00:56 "오늘 해야 할 일을 볼 겁니다… 몇 개가 있는지". 업무 보드는 **큰 버튼(뷰 버튼 2개 정도 들어갈 크기)**으로 진입. (김민주 10:07~10:24)
2. **진행 전/중/후 카운트**. 총 건수가 아니라 단계별로: 예) 총 7건 중 진행 전 5·중 1·후 0. 각 단계 카운트를 누르면 그 단계의 케이스를 볼 수 있다. (김민주 07:53~08:36)
3. **우선순위 기준 = 근거**. 김주용 13:13 "기준은 근거예요." 우선순위 산정의 축은 케이스가 가진 근거(위험·긴급도 등)다. (이승보 13:04 "우선순위 기준을 잡아야 되겠어")
4. **케이스 클릭 → 하위 에이전트 배치가 아래로 쫙**. 첫 케이스를 누르면 그 케이스에 배치될 에이전트들이 나온다. (김민주 01:12) **계층 구분**: 위=이 케이스에 대한 정보, 아래=에이전트·API에 대한 정보로 나눠 헷갈리지 않게. (김민주 09:03~09:19, 김주용 09:19 "계층을 이렇게 나누는 거고, 헷갈리지 않게")
5. **에이전트 실행 구조(역할 중심)**. 케이스 담당 에이전트 A가 있고(저희는 역할 중심), 하위 에이전트들이 저희가 만든 **스킬·MCP·플러그인·내부 규정**을 불러와 처리 → 각자 **MD 파일** 생성 → 담당 에이전트가 **취합해 최종 MD 1개**(RM이 해야 할 것)를 만든다. (김주용 02:04~03:26, 이승보 05:08~05:42)
6. **엔터-퍼스트 밀어내기 + 키보드 전용**. 케이스에서 엔터 → "너 들어가"(로딩) → 담당자는 다음 케이스로. 각 하위 에이전트를 엔터로 "통과, 통과" 승인하면 스트로크(강조)→딤드→아래로 내려가고 다음 에이전트가 올라온다. **마우스 절대 금지**(김민주 17:25 "왜냐면 한 손에는 노트, 한 손에는 이걸 움직여야 돼"), 방향키·스페이스·**숫자키(예: 2)로 열기**도 가능. (김주용 04:55, 06:39, 김민주 17:11~17:38)
7. **엔터에 과한 책임 금지**. 김민주 05:02 "엔터에 너무 큰 책임을 주면 안 돼요, 이 엔터 하나에." → 위험 결정은 2단 확인/추가 키. (§2:25 회의 FR-06 고위험=클릭승인 강제와 정합)
8. **각 에이전트 표시 = 붙는 이유(한 줄) + 기대값 + 실행/승인 안내**. 김민주 04:26 "이 에이전트를 쓰는 어떤 것을 연구할 건지 간단한 한 줄… 이런 것을 분석하려고 하니까 승인해 주세요", 15:30~16:20 "얘가 붙는 이유… 기대값… 엔터를 눌러 실행하세요."
9. **로딩·비동기·알림**. 엔터 즉시 로딩 표시(퍼센트 불필요, "그냥 로딩"). LLM 딜레이가 있으니 **백그라운드로 돌리고 다른 케이스를 보다가**, 완료되면 **알림(띠링)** → 해당 케이스로 돌아가 확인·승인. 예: 3번까지 진행하다 1번 완료 알림 → 1번 돌아가 승인 → 2번. (김주용 12:25~13:04, 김민주 07:27, 18:28~19:11)
10. **완료 케이스 = 어두운 색**. 끝난 케이스는 어둡게, 나머지는 진행 중 표시. (김민주 19:02)
11. **결과물 뷰 = MD 문서**. 에이전트들이 MD로 깔끔히 정리(노션 문서처럼 예쁘게). **통합본 MD + 개별 에이전트 MD**. 열람 방식: 옆에서 보기 / 탭 / **플로팅 모달(뒤 어두워지고 문서만 위에)**. (김민주 01:39, 19:24~21:20, 이승보 20:41~21:05)
12. **옵시디언/노션식 링크 시스템**. 통합 MD 안에 개별 MD들이 **링크**로 걸려, 눌러 타고 들어가 개별 문서를 본다(옵시디언·노션처럼). `@에이전트`(골뱅이 멘션)를 누르면 **그 에이전트 화면**으로 이동. 사용 스킬·출처는 문서 **맨 아래**에 쫙. (김주용 21:46~22:20, 김민주 19:29~19:46)
13. **딤드/블러 + 화살표(스크롤 인식)**. 화면 하단 잘리는 부분(노트북 화면 경계)에 살짝 블러/딤드 + 화살표로 "더 있음"을 인식시킨다. (김민주 09:54~10:07)
14. **RM 실제 업무는 사람이**. 취합 MD의 근거를 바탕으로 담당자가 실제 업무(전화·수기 보고서·신청 등)를 하고, 잘됐으면 승인(케이스 닫힘)·아니면 반려(재순환). 환각 최소화는 **RAG 내부지식·API 근거 + 하네스**로. (김주용 03:41~04:26, 06:11~06:36)
15. **시연 범위 = 2개 계열사만**. 김민주 22:28 "이거 두 개만 이렇게 만듭니다. 두 개 많이 하죠." (2:25·6/30 회의의 전북은행 + JB우리캐피탈 2계열사와 정합)
16. **제품 가치 포지셔닝**. "이게 시스템 안으로 들어오면 사실상 **납품할 수 있는 제품**"(김주용 12:01), "이것만 되면 무조건 3점"(김민주 12:17 — 심사 만점 기대). 단, 토큰 소모 큼(이승보 10:58~11:05 "토큰 진짜 많이 써요… 아직 코덱스·플로우도 안 쓰고 있어요").
