# 전세사기 보호 담당자 역할 하네스 (2026-07-03)

핵심 원칙: **"역할 = 화면 필터"가 아니라 "역할 = 독립 업무 하네스"**.
역할 레일의 "전세보호 담당자"는 더 이상 라벨-only 대시보드로 이동하지 않고,
자기 메뉴·데이터·AI·승인·감사 흐름을 가진 별도 운영 콘솔(`jeonseFraudProtectionHarness`)로 진입한다.
같은 패턴으로 RM·기업여신·FDS 역할도 각각의 하네스로 확장할 수 있다.

- roleKey: `jeonse-protection-officer` · workspaceId: `jeonse-protection`
- route base: `#/roles/jeonse-protection` (새로고침 유지, 케이스 상세 `/cases/:caseId`)
- activeView 마커: `jeonse-protection-harness`

## 1. 아키텍처 (계열사 하네스와 동일 패턴)

| 파일 | 레이어 | 역할 |
| --- | --- | --- |
| `app/jeonseProtection.config.js` | config | route/nav/업무유형 10종/위험신호 10종/라벨/공식근거 |
| `app/jeonseProtectionAgents.registry.js` | registry | 10개 전용 에이전트, 하네스 정책, `routeJeonseProtectionCase` |
| `app/jeonseProtection-db.js` | repository | role scope 강제 mock DB(`jpo-ops-db-v1`), `jpoRepository` |
| `app/jeonseProtectionServices.js` | service | 21개 count/PII 차단 검색/triage/생성/실행 기록/KPI |
| `app/jeonseProtection.helpers.js` | view-shared | 상태 pill·테이블·상세 패널·브레드크럼 |
| `app/jeonseProtection.view.*.js` | view | board / 21개 목록 / 5단계 위저드 / 에이전트 하네스 |
| `app/jeonseProtection.sidebar.js` | binding | 사이드바 점유/복원(계열사 하네스와 점유 중재), 검색, 카운트 |

공통으로 재사용하는 것은 presentation(4-zone 셸, `jbwc-*` CSS 토큰, `escapeHtml`/`iconSvg`)뿐이며,
business 상수/함수(`jbwcTable`, `JBWC_*`, 메인 `agents`, `roleDashboardPage`)는 verify 게이트에서 **금지 문자열**로 차단된다.

## 3. 업무 유형 10종 → 파생 테이블

계약 전 위험 점검→risk_assessments(preContract) · 전세가율·시세→price_ratio_checks ·
권리관계·등기→registry_checks · 보증보험/HUG→guarantee_reviews(사람 검토 고정) ·
경공매 지원→referrals(auction, 사람 검토) · 피해자 결정 보조→victim_support_reviews(사람 검토) ·
법률상담→referrals(legal, 사람 검토) · 심리·주거→referrals(care) ·
취약고객 보호→risk_assessments(vulnerableTenant, 사람 검토) · 긴급 위험 알림→jeonse_alerts(에스컬레이션 필수).

신규 접수(5단계 위저드) 저장 시 공통 기록: jeonse_cases + jeonse_tasks + audit_logs(JPO_CASE_CREATED)
+ ai_analysis_requests + agent_runs(+handoffs) + 위 파생 테이블 + (사람 검토/고위험 시) approvals.

## 4. 에이전트 10종과 라우팅 규칙

Jeonse Triage Orchestrator / Price Ratio & Market Signal / Registry Rights / Guarantee & HUG Support /
Auction·Public Sale Support / Victim Decision Checklist / Legal·Psychological Referral /
Tenant Communication Draft / Privacy & Evidence Redaction / Audit & Approval (각각 한국어 displayName 보유).

키워드/신호 라우팅: 전세가율·시세→price · 근저당/압류/신탁/등기→registry · 보증보험/HUG→guarantee(사람 검토) ·
경매/공매/우선매수/낙찰/퇴거→auction(high 이상+사람 검토) · 피해자 결정/특별법/신청서류→victim(사람 검토) ·
법률상담/심리/긴급주거→referral · 안내문/콜백/문자→comms(**항상 approval pending**) ·
개인정보/증빙/외부반출→privacy · high/critical→escalation 필수+audit 핸드오프.

## 5. Guardrails (공통 blockedActions + 정책)

전세사기 여부/피해자 결정/보증 가입/법률 자문 **확정 금지** · 신청 제출/대행 금지 ·
대출 승인/금리/한도 판단 금지 · 개인정보 원문 저장/출력 금지(익명 Ref만: TENANT-REF-*/PROPERTY-REF-*/
LANDLORD-REF-*/ADDRESS-REF-*, 보증금은 구간값) · 고객 자동 발송 금지 · high/critical 자동 종결 금지 ·
모든 output "내부 운영 참고용 · 담당자 검토 필요" · 검색에서 주민번호/전화/계좌형 패턴 차단.

## 6. 공식 근거 (안내 후보로만 표시)

국토교통부 전세사기피해자 지원관리시스템(molit.go.kr) · HUG 전세피해지원센터(khug.or.kr) ·
전세사기피해자 지원 및 주거안정에 관한 특별법(law.go.kr) · 금융위원회 금융지원 프로그램(fsc.go.kr).
법령·지원요건은 변동 가능하므로 화면에 항상 "최신 기준 담당자 확인 필요"를 표시하며,
AI는 신청 가능 여부를 단정하지 않는다.
