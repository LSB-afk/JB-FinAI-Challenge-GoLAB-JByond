# 팀 GoLAB

> **GoLAB** — 사람 4명이 AI 에이전트 팀 하나를 지휘해, 25일간 금융 AI의 **운영 체계**를 설계·구현한 팀. 제품이 주장하는 운영 계약(케이스 → 에이전트 → 사람 승인 → 감사 기록)을 개발 과정 자체에 그대로 적용했다.

| 이름 | 역할 | 주요 작업 |
|---|---|---|
| 이승보 | 개발 (프로덕트 엔지니어링) | 제품 코드 정본(JB_project2) 설계·구현 · 역할/계열사 콘솔·백엔드 승격 · fork→PR 리뷰·머지 게이트 운영 |
| 이재형 | 개발 (AI 오케스트레이션) | 멀티세션·서브에이전트 병렬 개발 지휘 · LLM 게이트웨이·3계층 메모리·감사 해시체인 · 운영 볼트(SSOT·결정 로그) 설계 |
| 김주용 | PM | 일정·범위·의사결정 조율 · 3도메인(여신·전세·보이스피싱) 문제 정의 · 심사 Q&A 방어 체계 |
| 김민주 | 디자인 | 발표 덱(JByond 리브랜딩) · 콘솔 UX(keyboard-first·Enter Interaction) 컨셉 |

## AI 협업 방식

지휘는 사람, 실행은 병렬 AI로 나눴다. 여러 세션과 서브에이전트를 동시에 돌리되 외부 제출·공개·고객 대상 결정은 반드시 사람이 승인했고, 모든 의미 있는 작업은 로그로 남겼다(Capture-by-default). 멀티 에이전트 오케스트레이션을 개발 방법론으로 실증한 것이 이 팀의 강점이다.
